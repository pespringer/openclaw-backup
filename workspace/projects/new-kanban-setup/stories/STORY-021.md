# STORY-021 — Add decision log panel to Mission Control

## Status
Backlog




## Story
As a user, I want a visible decision log in Mission Control, so that important choices are easy to review without digging through story notes and markdown files.


## Why this matters
- Reduces repeated debates.
- Makes prior reasoning easy to recover.
- Improves continuity when work resumes after time away.

## Acceptance Criteria
- [ ] Decisions can be captured in a structured format.
- [ ] Mission Control shows recent decisions in a dedicated panel.
- [ ] Decisions link back to relevant stories or docs.
- [ ] The panel remains lightweight and readable.

## Projected Work
- Estimate: M
- Approach:
  1. Define a minimal decision record format.
  2. Add a decision log source file or section.
  3. Render recent decisions in the UI.
  4. Link decisions to stories/projects.
- Dependencies: Documentation Hub patterns
- Risks: duplicating too much story content instead of summarizing.

## Deliverable
- Decision log panel and supporting storage format.
