# STORY-010 — Add Documentation Hub to Mission Control

## Status
Done



## Story
As an operator, I want Mission Control to surface important docs and runbooks, so that the system is easier to operate and maintain without context switching.


## Why this matters
- Keeps critical reference material close to execution.
- Reduces search friction.
- Makes Mission Control more useful as a real workspace.

## Acceptance Criteria
- [ ] Documentation section exists in the UI.
- [ ] Key docs are surfaced clearly.
- [ ] Runbooks and architecture notes are easy to access.
- [ ] The hub supports future expansion.

## Projected Work
- Estimate: M
- Approach:
  1. Identify key docs to surface first.
  2. Add doc metadata and grouping.
  3. Render the hub in UI.
  4. Preserve room for deeper document navigation later.
- Dependencies: health and agent sections defined
- Risks: surfacing too much documentation at once.

## Deliverable
- Documentation Hub integrated into Mission Control.
